## [0:00 - 2:00] 차별점 분석 (팀원D)

---

안녕하세요. TeamTeam 팀입니다.

저희 서비스는 대학교 팀 프로젝트 협업을 위한 클라우드 네이티브 플랫폼입니다.

기존 협업 도구들과 비교했을 때 세 가지 핵심 차별점이 있습니다.

첫째, **AI 일정 자동 추천**입니다.
Notion이나 Jira는 일정을 사람이 직접 입력해야 합니다.
저희는 Gemini 2.0 Flash API를 연동해서, 팀 상황을 분석하고 일정을 자동으로 제안합니다.
팀원들은 제안을 승인하거나 거절하기만 하면 됩니다.

둘째, **서비스 내 통합 실시간 채팅**입니다.
보통 Slack이나 카카오톡 같은 별도 메신저를 써야 하는데,
저희는 프로젝트 관리와 채팅이 하나의 서비스 안에 통합되어 있습니다.

셋째, **팀원 상호 평가 기능**입니다.
프로젝트가 끝나면 팀원들끼리 익명으로 기여도를 평가할 수 있습니다.
기존 협업 도구에는 없는 기능입니다.

그리고 가장 중요한 차별점으로, **저희 서비스는 AWS 위에 직접 구성된 클라우드 네이티브 아키텍처**로 운영됩니다.
Notion이나 Jira는 블랙박스 SaaS지만, 저희는 인프라 전체를 직접 설계하고 모니터링합니다.

## [2:00 - 5:00] 아키텍처 + 4 Ops (팀원D)

---

아키텍처를 설명드리겠습니다.

전체 구조를 보시면,
사용자는 CloudFront를 통해 S3에 배포된 React 앱에 접속하고,
API 요청은 WAF가 붙은 ALB를 거쳐 두 개의 가용 영역에 있는 EC2로 전달됩니다.

**이중화** 측면에서는,
ALB가 ap-northeast-2a와 ap-northeast-2c 두 AZ에 걸쳐 트래픽을 분산하고,
Auto Scaling Group이 장애 발생 시 자동으로 인스턴스를 교체합니다.
발표 중에 실제로 인스턴스 하나를 종료해도 서비스가 계속 동작하는 것을 보여드리겠습니다.

**DevOps** 측면에서는,
GitHub main 브랜치에 코드를 push하면,
GitHub Actions가 자동으로 Docker 이미지를 빌드해서 ECR에 올리고,
EC2에 무중단으로 배포됩니다.

**SRE** 측면에서는,
FastAPI 내부에 9개의 커스텀 Prometheus 메트릭을 심어두었고,
Grafana 대시보드로 AI 응답시간, WebSocket 접속자 수, HTTP 에러율 등을 실시간으로 확인합니다.
알림 룰이 7개 정의되어 있어, 예를 들어 AI 응답시간이 p95 기준 5초를 넘으면 이메일로 알림이 갑니다.

**SecOps** 측면에서는,
API Key는 AWS Secrets Manager에 보관하고, EC2는 IAM 역할을 통해서만 접근합니다.
ALB 앞에 WAF를 붙여 SQL Injection과 XSS를 차단하고,
CloudTrail로 모든 AWS API 호출을 감사합니다.

**FinOps** 측면에서는,
프론트엔드를 S3에 올리는 것만으로도 EC2 대비 월 비용이 약 95% 절감됩니다.
AWS Budgets를 설정해서 팀 계정이 40달러를 넘으면 이메일 알림을 받습니다.

그럼 실제 서비스를 시연하겠습니다.

## [5:00 - 13:00] 데모 시연

---

### Step 1 — 로그인 + 대시보드 (1분)

지금 브라우저에서 CloudFront URL로 접속하면,
S3에 배포된 React 앱이 로드됩니다.

로그인을 하면 팀 대시보드가 보입니다.
상단에는 전체 업무 진행률,
왼쪽에는 오늘 마감 업무,
오른쪽에는 팀원 목록이 표시됩니다.

이 화면에서 보이는 데이터는 모두 FastAPI가 Supabase PostgreSQL에서 실시간으로 가져오는 것입니다.

---

### Step 2 — AI 일정 추천 (1분 30초)

저희 서비스의 핵심 기능인 AI 일정 추천을 보여드리겠습니다.

일정 페이지로 이동하겠습니다.

AI 추천 버튼을 클릭하면,
FastAPI가 현재 팀의 업무 현황을 분석해서 Gemini 2.0 Flash API에 전달하고,
AI가 최적화된 일정을 제안합니다.

보시는 것처럼 업무별로 추천 일정이 생성되었습니다.
각 항목을 승인하면 실제 일정에 반영되고,
거절하면 다시 제안을 받을 수 있습니다.

단순히 API를 호출하는 것에서 그치지 않고,
AI 응답시간을 Prometheus 히스토그램으로 측정해서 SRE 대시보드에 바로 반영됩니다.

---

### Step 3 — 실시간 채팅 (1분)

이제 실시간 채팅 기능입니다.

채팅 페이지로 이동하겠습니다.

브라우저 두 개를 열어서 서로 다른 계정으로 접속하면,
한쪽에서 메시지를 보내면 polling 없이 WebSocket으로 즉시 상대방 화면에 나타납니다.

현재 접속자 수도 ws_chat_active_connections 메트릭으로 실시간 추적됩니다.

---

### Step 4 — 업무 관리 + Grafana 실시간 연동 (1분)

업무 관리 페이지입니다.

상태 버튼을 클릭해서 '진행 전' → '진행 중' → '완료'로 변경해보겠습니다.

이제 Grafana 대시보드로 이동하면,
task 관련 메트릭 수치가 실시간으로 바뀐 것을 확인할 수 있습니다.

사용자의 행동 하나하나가 즉시 SRE 메트릭에 반영되는 것입니다.

---

### Step 5 — 공지사항 + 자료실 (1분)

공지사항 페이지입니다.
팀장 공지는 상단에 핀으로 고정되어 표시됩니다.

자료실에는 Google Drive나 GitHub 링크를 등록해서 팀원들이 바로 접근할 수 있습니다.

---

### Step 6 — Grafana 대시보드 라이브 (1분 30초)

Grafana 대시보드입니다.

첫 번째 패널은 AI 일정 추천 응답시간 히스토그램입니다.
방금 시연한 AI 호출의 응답시간이 기록되어 있습니다.

두 번째 패널은 WebSocket 접속자 수입니다.
채팅방에 접속한 인원이 실시간으로 표시됩니다.

세 번째 패널은 HTTP 에러율입니다.

알림 룰을 보시면 7개가 정의되어 있습니다.
예를 들어, AI 응답시간 p95가 5초를 넘으면 critical 알림이 발송됩니다.
이 알림은 SNS를 통해 이메일로 전달됩니다.

---

### Step 7 — CI/CD 파이프라인 (1분)

마지막으로 DevOps 파이프라인입니다.

GitHub Actions 탭을 보시면,
최근 배포 워크플로가 실행된 것을 확인할 수 있습니다.

흐름은 이렇습니다.
main 브랜치에 push → Docker 이미지 빌드 → ECR에 push → EC2에서 pull → 컨테이너 재시작.

ALB 헬스체크 덕분에 새 컨테이너가 준비되기 전까지는 기존 컨테이너가 트래픽을 받아서 무중단 배포가 됩니다.

## [13:00 - 15:00] 마무리 (팀원D)

---

정리하겠습니다.

저희 TeamTeam은 단순한 협업 도구가 아니라,
**AWS 클라우드 네이티브 아키텍처 위에 구축된 서비스**입니다.

- DevOps: GitHub Actions로 코드 push만 하면 자동 배포
- SRE: Prometheus + Grafana로 9개 메트릭 실시간 모니터링, 7개 알림 룰
- SecOps: Secrets Manager, WAF, CloudTrail로 3중 보안
- FinOps: S3 정적 호스팅으로 서버 비용 95% 절감, AWS Budgets으로 비용 통제

각 영역을 단순히 이론으로 설명하는 것이 아니라,
실제로 동작하는 서비스에 모두 적용했습니다.

감사합니다.
